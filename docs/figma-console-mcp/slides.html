The problem

Design systems leak at the seams

  • Figma variables drift from tokens/tokens.json
  • Variants exist in one place but not the other
  • Mocks reinvent Navbar / SideNav / PageHeader shells
  • Handoff becomes archaeology instead of a checklist

What MCP is

A USB-C port for AI tools

Model Context Protocol lets Cursor (or Claude, etc.) call structured tools — not just guess from screenshots.

  • You stay in natural language
  • The server reads/writes Figma with real APIs
  • Beam repo in Cursor = code context for free

Important distinction

Two different “Figma MCPs”

Official Figma MCP

Figma’s own server / Cursor plugin

  • Design → code, Code Connect
  • Selection-aware Dev Mode flows
  • Great for implementation

Figma Console MCP

Open source (Southleft) — today’s focus

  • Tokens, variables, DS kit dump
  • Create / arrange components
  • A11y + design–code parity

You can run both. Today we deepen Console MCP.

Install · Part 1

Prerequisites

  1. Node.js 18+ (node --version)
  2. Figma Desktop (not only the browser)
  3. Cursor installed
  4. Figma personal access token (figd_…)

Install · Part 2

Wire Cursor → MCP

Settings → Tools & MCP → Add Custom MCP, or edit ~/.cursor/mcp.json

{
  "mcpServers": {
    "figma-console": {
      "command": "npx",
      "args": ["-y", "figma-console-mcp@latest"],
      "env": {
        "FIGMA_ACCESS_TOKEN": "figd_…",
        "ENABLE_MCP_APPS": "true"
      }
    }
  }

Install · Part 3

Desktop Bridge plugin

  1. Open Beam DS file in Figma Desktop
  2. Plugins → Development → Import plugin from manifest…
  3. Select ~/.figma-console-mcp/plugin/manifest.json
  4. Run Figma Desktop Bridge and leave it open

This WebSocket bridge is what unlocks create / edit / variables.

Use case · Tokens

Figma variables vs tokens.json

Export variables from the Beam Figma file as DTCG JSON.
Compare to tokens/tokens.json under beam.color.semantic.
Report missing / extra / renamed — do not write files yet.

Beam still validates with yarn validate:tokens. MCP is the bridge; JSON remains the codegen source of truth.

Use case · Components

Spec a component for handoff

Get the Button component from Beam Figma with a visual reference.
Compare to src/components/Button.tsx — what exists, what’s Figma-only,
and what would need a new prop (flag new props as discouraged).

Use case · Parity

Design–code scorecard

Run design-code parity for SideNav: Figma vs src/components/SideNav/.
Scored report + concrete fix items for design and code.
Prefer semantic tokens over hard-coded hex.

Best when this repo is open in Cursor alongside the Figma bridge.

Use case · Creation

Scaffold variants — draft only

On a scratch page, draft component set "FilterChip"
(selected × size). Use existing Beam variables — no new hex.
Name it "[MCP] FilterChip — review". Do not publish to the library.

Use case · Quality

A11y + layout shells

  • A11y: contrast, targets, truncation on a selected frame
  • Layouts: map mocks to EnvironmentBanner → Navbar → SideNav → PageHeader
  • Stops one-off chrome that fights Beam’s layout contracts

Use case · Collaboration

FigJam workshops & changelogs

  • Token-naming boards with columns for Figma name / --b-* / ships?
  • Version-history summaries after a DS refresh
  • Generated markdown docs merging Figma + Storybook sources

Prompt craft

Constraints beat vibes

Do

  • File URL + component name
  • “Existing tokens only”
  • “Draft / don’t publish”
  • “Report first, then apply”
  • Beam few-props philosophy

Don’t

  • “Make it modern”
  • Silent library publishes
  • Blind token imports
  • Five new style props
  • Paste PATs into chat logs

Gotchas

When it misbehaves

  • Bridge not running → writes silently fail or status looks empty
  • Read-only SSE path → can’t create (upgrade to NPX)
  • After MCP upgrades → re-import plugin manifest if tools break
  • First call slow → npx downloading @latest
  • AI drafts still need a human Beam review
